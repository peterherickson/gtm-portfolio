OpenClaw

An AI infrastructure system that lives in my workspace and actually does things. Not a chatbot — an agent with access to files, tools, and the ability to take action.

Outputs advanced websites, research synthesis, and organized knowledge that would otherwise be difficult to create or maintain. Built this portfolio, analyzed my LinkedIn network across 10+ years, and compiled the Philly Artist Census visualizations.

The Philosophy

Most AI tools are built to chat. OpenClaw is built to work. It has access to my files, my calendar, my messages, and my systems. It can read, write, search, and execute tasks — not just talk about them.

The goal isn't to have a conversation. It's to have a partner who can actually help with the work. Someone who remembers what matters, takes initiative on internal tasks, and asks before doing anything external.

I'm not trying to automate myself out of existence. I'm trying to offload the cognitive overhead so I can focus on the work that actually requires me.

Technical Stack

→Docker containers for isolated tool execution and workspace management
→Ollama for LLM inference — Qwen3.5 35B running locally, Qwen3.5 397B via Ollama Cloud for heavier tasks
→WebSockets for real-time communication between gateway and agents
→Node.js runtime with persistent memory and file system access
→WhatsApp integration for async messaging and control

OpenClaw is both a tool I use and a project I'm actively developing. The system evolves as I identify new workflows to automate.
